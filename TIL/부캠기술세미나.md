### 권혁우 그래프 ql
- 클라이언트가 api서버에 요청하는 쿼리 언어
- 클라이언트는 서버에서 데이터를 받아서 랜더링
    - 서버에서 json을 받아온다.
그래프ql쿼리/api
- api는 요청 데이터, ql은 틀을 스키마를 통해 데이터를 파싱해서 가져올 수 있다.

그래프ql스키마
- 서버가 제공할 수 있는 데이터의 형태를 정의
    - 타입이 존재한데
- !는 notnull과 같음
루트 타입
- 작업을 명령하는것
- query(데이터조회), mutation(데이터수정, 삽입, 삭제), subscription(클라이언트 서버와 실시간 통신할때)

rest api 문제
- 불필요한 데이터를 함께 받아옴 - 데이타오버패칭
    - ql을 통해 쿼리를 이용하면 이러한 문제를 해결 가능
- 여러 리소스를 가져와서 랜더링 해야함, - 여러 데이터에 대해 조합을 해야하는 경우가 있음
    - ql은 필요한 데이터를 하나에 몰아서 작성가능하데

장점
- 자동문서관리
- 효율적인 데이터 요청
    - 클라이언트의 파싱하는 부분이 없어져 간결해짐
- 플랫폼별 데이터요청 쉬움

### 육진형 - (대충 도커 쓰자는 이야기)
- vm과다름
    - vm 하드웨어를 가상화, 독립된 공간, 성능상 안좋음
    - 도커 호스트로 격리된 실행환경 container를 가짐
- container
    - 어플과 라이브러리만 가짐, host의 자원을 공유
- cgroup
    - cpu같은 시스템 자원의 프로세스별 배분
- namespace
    - 프로세스, 네트워크 독립공간 제공
- image
    - 서버프로그래, 소스코드등 
    - 이게 실행하면 container
- 레이저 저장방식
    - 새로 저장시 기존꺼는 따로 가져와서 쓴다고 한다. 그림을 생각
    - 껐다 키면 지워진대

- 배포에 사용할수있는 개발 환경
